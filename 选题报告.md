# 山海镜 —— 基于知识图谱与RAG增强的多智能体地理信息检索系统

## 选题报告

---

## 一、选题目的

### 1.1 选题背景

当前高中生在学习地理知识时面临以下困境：

**1. 现有 AI 问答工具缺乏可视化。** 学生通过豆包、DeepSeek 等大模型查询地理问题（如"解释向斜背斜"），能得到文字描述，但无法直观看到该地质现象在地图上的实际位置和地形表现。纯文本回答导致学生只能机械记忆答案，难以建立空间理解和感性认知。

**2. 地理知识与地图脱节。** 传统地图应用（高德、百度地图）能定位地点，但不具备地理教学能力；AI 问答能解释概念，却无法联动地图进行空间演示。学生在"知道概念"和"看到现象"之间存在断层。

**3. 视频学习效率低下。** B 站、抖音等平台虽有大量地理教学视频，但质量参差不齐、内容散乱，学生需要耗费大量时间筛选，且需在视频 App 和问答工具之间反复切换，学习流程割裂。

**4. 缺乏交互式学习闭环。** 现有工具多为单向输出——要么给文字答案，要么播放视频——缺乏"提问→定位→演示→追问→直到理解"的完整学习循环。学生在不理解某个知识点时，无法就地追问、深入探究。

针对以上痛点，本文设计并实现**山海镜**——一个基于知识图谱与 RAG 增强的多智能体地理学习系统。该系统将大模型对话、地图可视化、视频检索与总结、语音交互整合为一体，为高中生提供"在地图上学习地理"的全新体验。

### 1.2 同类系统分析

| 类别 | 代表产品 | 优势 | 不足 |
|------|---------|------|------|
| 通用 AI 问答 | DeepSeek、豆包、Kimi | 文本回答能力强，覆盖知识面广 | 纯文本输出，无地图联动，无法可视化；回答偏理论，缺乏空间直观 |
| 地图应用 | 高德地图、百度地图、Google Earth | 定位精准，地图数据丰富 | 不具备地理教学能力，无法解释地质现象、分析地形成因 |
| 搜题工具 | 作业帮、小猿搜题 | 题库丰富，拍照识别快 | 偏重解题而非理解，缺乏互动追问，地理空间类题目展示不足 |
| 在线教育 | 学而思网校、猿辅导 | 课程体系完整，师资专业 | 一对多单向教学，无法个性化实时答疑，缺乏自主探索能力 |
| 视频平台 | B 站、抖音知识区 | 视频量大，部分内容优质 | 质量良莠不齐，筛选耗时；与学习工具割裂，需跳转观看 |

**山海镜的差异化定位**：上述系统分别在"问答""地图""搜题""教学""视频"某一维度上具有优势，但没有一个系统将**多智能体AI对话 + 地图联动定位 + 视频检索总结 + 语音交互**整合为一个完整的地理学习闭环。山海镜的核心创新在于：让学生在地图上直接"看见"地理知识，通过AI对话持续追问直到真正理解，并在应用内完成视频学习与总结。

## 二、系统分析

### 2.1 需求分析

#### 功能需求

| 编号 | 功能 | 描述 |
|------|------|------|
| F1 | 地理知识问答 | 用户输入地理问题，系统调用 LLM 生成回答，支持多轮追问 |
| F2 | 地图联动定位 | 识别问题中的地名，自动将地图飞行定位到该地点，添加标记与信息窗 |
| F3 | 地理实体识别 | 从用户问题中自动提取地理实体（山、河、湖、省、市等） |
| F4 | 知识图谱查询 | 返回地理实体的关系网络（属于哪个山脉、发源于何处、流经哪些省份等） |
| F5 | 景观图片展示 | 展示地理实体的实景图片，横向滑动浏览 |
| F6 | 视频检索与总结 | 根据地理关键词检索 B 站/抖音相关教学视频，内嵌播放，并自动生成视频内容摘要 |
| F7 | 语音输入 | 支持语音输入问题，方便学生快速提问 |
| F8 | 探索历史 | 保存最近 50 条探索记录，支持回看 |
| F9 | 知识图谱可视化 | 全屏 ECharts 力导向图展示实体关系网络，支持触摸互动 |

#### 非功能需求

| 编号 | 需求 | 描述 |
|------|------|------|
| NF1 | 响应时间 | 问答接口 p95 < 5 秒 |
| NF2 | 移动端适配 | 支持 iOS/Android 手机浏览器访问，PWA 可安装到桌面 |
| NF3 | 用户体验 | 界面友好，适合高中生操作习惯，降低学习门槛 |
| NF4 | 可扩展性 | 知识库可扩充（添加新 PDF 文献更新知识） |
| NF5 | 低成本运行 | 使用 DeepSeek 等国产低价 API，整体运行成本可控 |

### 2.2 可行性分析

#### 技术可行性

| 模块 | 技术方案 | 成熟度 | 风险 |
|------|---------|--------|------|
| LLM 对话 | DeepSeek API | 业界成熟 | 低：API 稳定可用 |
| 地图定位 | 高德地图 JS API 2.0 | 业界成熟 | 低：需申请 Key |
| 语音识别 | Web Speech API（浏览器原生） | 成熟，Chrome/Safari 支持 | 低 |
| 知识图谱 | NetworkX + 规则/LLM 三元组抽取 | 方案可行，规模可控 | 中：抽取精度依赖 Prompt 设计 |
| RAG 检索 | ChromaDB + BGE Embedding + BM25 | 业界成熟 | 低：小型知识库性能无忧 |
| 多智能体 | LangGraph Agent 编排 | 业界前沿 | 中：编排逻辑需调试 |
| PWA | Service Worker + Manifest | 业界成熟 | 低 |
| 视频内嵌 | B 站/抖音嵌入式播放器 | 方案可行 | 中：部分平台有跨域限制 |

**结论**：所有核心技术均有成熟方案或可行替代方案，技术风险可控。

#### 开发可行性

- **开发周期**：15 天，单人全栈开发
- **每日投入**：10 小时，总计 150 小时
- **技术栈覆盖**：Python（FastAPI）+ JavaScript（Vue 3）+ 多个第三方 API
- **风险缓解**：每个模块预留 1 天缓冲时间，关键路径优先实现

**结论**：在合理的时间管理与任务切分下，开发目标可达成。

#### 应用可行性

- **目标用户**：高中地理学生，对地理学习有可视化、互动化需求
- **使用场景**：课堂辅助学习、课后自主探索、考前知识点复习
- **推广方式**：PWA 无需安装，扫码即用，降低使用门槛

## 三、系统设计

### 3.1 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                    移动端 PWA（Vue 3 + Vant UI）               │
│  ┌─────────┐ ┌──────────┐ ┌──────────┐ ┌───────────────┐  │
│  │ 语音输入 │ │ 对话界面  │ │ 地图组件 │ │ 视频播放面板   │  │
│  └─────────┘ └──────────┘ └──────────┘ └───────────────┘  │
└──────────────────────────┬──────────────────────────────────┘
                           │ HTTP / SSE（流式）
┌──────────────────────────┴──────────────────────────────────┐
│                   FastAPI 后端（Python）                     │
│  ┌─────────────────────────────────────────────────────┐   │
│  │          多智能体协作层（LangGraph 编排）              │   │
│  │                                                     │   │
│  │  🎯Orchestrator → [🌍Geo ‖ 🔗KG ‖ 📄RAG ‖ 🎬Video]  │   │
│  │                → ✍️Synthesis → 🛡️Critique           │   │
│  └─────────────────────────────────────────────────────┘   │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐    │
│  │ 地理编码  │ │ KG 查询  │ │ RAG 检索 │ │ 视频检索  │    │
│  │(高德API) │ │(NetworkX)│ │(ChromaDB)│ │(B站/抖音) │    │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘    │
└──────────────────────────────────────────────────────────┘
                           │
┌──────────────────────────┴──────────────────────────────────┐
│                   数据层                                     │
│  PDF 文献 → 解析 → NER → 三元组提取 → KG 构建 + RAG 索引    │
│  地图数据（高德API） ｜ 视频元数据 ｜ 景观图片库               │
└──────────────────────────────────────────────────────────┘
```

**数据流**：
1. 用户通过语音或文字提问
2. Orchestrator Agent 分析查询意图，分解任务
3. GeoAgent、KGAgent、RAGAgent、VideoAgent 并行执行各自子任务
4. SynthesisAgent 融合所有结果，生成回答
5. CritiqueAgent 核查事实准确性
6. 前端渲染：地图定位 + 知识卡片 + 景观图片 + 视频面板 + 对话回答

### 3.3 数据库设计

#### 知识图谱 Schema

**实体类型（Node Types）**：

| 类型 | 核心属性 | 示例 |
|------|---------|------|
| MountainRange | name, aliases, elevation_max, length_km, provinces | 秦岭 |
| Mountain | name, aliases, elevation, lat, lon, parent_range, province | 太白山 |
| River | name, aliases, length_km, basin_area_km2, provinces | 长江 |
| Lake | name, aliases, area_km2, elevation, province, lat, lon | 青海湖 |
| Province | name, abbreviation, region, capital | 陕西省 |
| GeologicalStruct | name, type, area_km2, age_period, lat, lon | 青藏高原 |
| ClimateZone | name, type, avg_temp, avg_precipitation | 亚热带季风气候 |

**关系类型（Edge Types）**：

| 关系 | From → To | 示例 |
|------|-----------|------|
| LOCATED_IN | 山/河/湖 → 省/市 | 太白山→陕西省 |
| BELONGS_TO | 山 → 山脉 | 太白山→秦岭 |
| FLOWS_THROUGH | 河 → 省/市 | 长江→四川省 |
| ORIGINATES_AT | 河 → 山脉/构造 | 汉江→秦岭 |
| TRIBUTARY_OF | 河 → 河 | 汉江→长江 |
| FLOWS_INTO | 河 → 湖/海/河 | 长江→东海 |
| ADJACENT_TO | 山脉 ↔ 山脉 | 秦岭↔大巴山 |

#### 向量数据库（ChromaDB）

| Collection | 存储内容 | 用途 |
|-----------|---------|------|
| geo_knowledge | PDF 文献分块 + Embedding | RAG 文档检索 |
| video_metadata | 视频标题/描述/标签 + Embedding | 视频语义搜索 |

#### 本地存储

| 数据 | 格式 | 用途 |
|------|------|------|
| 景观图片 | JPG/PNG 文件 + image_index.json | 实体名→图片路径映射 |
| 知识图谱 | GraphML 文件 | NetworkX 序列化 |
| 用户历史 | localStorage（浏览器端） | 探索记录 |


### 3.5 技术选型

| 层 | 技术 | 选型理由 |
|----|------|---------|
| **多智能体框架** | LangGraph | 状态图编排，支持条件分支+并行执行 |
| **后端框架** | FastAPI (Python) | 异步高性能，自动 API 文档，Pydantic 校验 |
| **LLM** | DeepSeek API | 中文最优，40-60 倍便宜于 GPT-4o，国内直连 |
| **前端框架** | Vue 3 (CDN) + Vant UI 4 | 移动端组件库，MVP 阶段 CDN 零配置 |
| **地图** | 高德地图 JS API 2.0 | 中国地理最详细，GCJ02 坐标系 |
| **语音输入** | Web Speech API | 浏览器原生支持，无需第三方 |
| **知识图谱** | NetworkX + GraphML | 零运维，轻量级，适合小规模 KG |
| **向量数据库** | ChromaDB | 零配置，Python 原生，嵌入式部署 |
| **Embedding** | bge-small-zh-v1.5 | 中文语义最优，仅 200MB 显存 |
| **稀疏检索** | rank-bm25 + jieba | 中文关键词精确匹配 |
| **视频检索** | B 站/抖音开放 API | 内嵌播放器，无需跳转 |
| **部署** | Docker + Nginx | 容器化，一次配置到处运行 |
| **PWA** | Service Worker + Manifest | 可安装到手机桌面，离线缓存 |
